## 1. 미디어 통계의 함정: 우리는 왜 숫자에 속는가?

### 숫자가 주는 '객관성의 **환상**'

대중은 텍스트로 된 주장에는 의심을 품지만, '숫자'나 '그래프'가 제시되면 무의식적으로 이를 과학적이고 객관적인 '팩트(Fact)'로 받아들이는 경향이 있다. 언론과 이익집단은 이 심리를 적극적으로 이용한다.

> **[경고] 기계적 **중립**(Mechanical Neutrality)의 함정**
>
> 과학적으로 명백히 검증된 사실과 조작된(혹은 빈약한 표본의) 주장을 50대 50의 비중으로 나란히 보도하여, 마치 두 주장이 동등한 가치를 지닌 것처럼 대중을 기만하는 행위이다. 이는 진실을 외면한 허울뿐인 객관성이다.

### 언론이 통계를 왜곡하는 구조적 이유

- **주장의 **정당화**:** 특정 정치적 성향이나 경제적 이익을 대변하기 위해, 결론을 미리 정해놓고 데이터만 끼워 맞춘다.
- **클릭 **저널리즘**(Clickbait):** "미세한 변화"로는 시선을 끌 수 없으므로 "폭등", "충격" 등의 자극적인 단어와 함께 수치를 과장한다.

**결론:** 통계적 리터러시가 없다면, 우리는 누군가 편집한 '가짜 현실'을 진짜로 착각하는 수동적 소비자로 전락한다.

## 2. 통계 왜곡의 고전적 기법 1

### 1) 체리피킹(Cherry Picking)

전체 데이터 흐름을 보여주지 않고, 자신의 주장을 뒷받침할 수 있는 특정 '기간'이나 '부분'만 잘라내어 제시하는 기법이다.

> **사례:** 10년간 꾸준히 300% 우상향한 주가 그래프에서, 일시적으로 5% 하락했던 특정 6개월만 잘라내어 "경영 위기 심각"이라고 보도한다.

### 2) 그래프 **축**(범위) 삭제 조작

막대그래프 등의 Y축 시작점을 '0'이 아닌 다른 숫자로 조작하여, 아주 미미한 차이를 거대하게 부풀리는 시각적 사기 기법이다.

> **사례:** 지지율 40%와 42%의 접전 상황에서 Y축을 39%부터 시작하게 설정한다. 42% 막대가 40% 막대보다 3배 더 커 보이는 심각한 착시를 일으킨다.

---

## 3. 통계 왜곡의 고전적 기법 2

### 3) 표본 추출의 **오류**(Sampling Bias)

전체 모집단을 대표할 수 없는, 심하게 편향된 소수의 사람들에게만 의견을 묻고 이를 "전체 집단의 뜻"인 것처럼 포장하는 기법이다.

> **1936년 미 대선 리터러리 다이제스트 참사**
>
> 잡지사는 240만 명의 막대한 표본으로도 선거 결과를 틀렸다. 표본을 '전화번호부'와 '자동차 소유주'(당시 부유층) 명부에서만 추출하여, 공화당 지지자에게 극도로 편향되었기 때문이다. 반면 갤럽은 인구 비율을 맞춘 단 5만 명으로 결과를 정확히 예측하였다.

### 4) 심슨의 **역설**(Simpson's Paradox)

데이터를 뭉뚱그려 볼 때의 결과와, 세부 그룹으로 쪼개어 볼 때의 결과가 180도 정반대로 뒤집히는 통계적 착시 현상이다.

> **UC 버클리 성차별 논란**
>
> 전체 대학원 합격률은 남성 44%, 여성 35%로 여성이 차별받는 듯 보였다. 그러나 학과별로 분석하자 대부분 여학생 합격률이 더 높았다. 여학생들이 애초에 경쟁률이 극도로 높은 인문계열에 대거 지원했기 때문에 벌어진 '전체 평균의 함정'이다.

---

## 4. 통계 왜곡의 고전적 기법 3

### 5) 상관관계와 인과관계의 혼동

두 변수가 우연히 같이 움직이는 '상관관계'일 뿐인데, 미디어가 이를 "A가 B의 원인이다"라는 '인과관계'로 비약하는 오류이다.

> **사례:** "여름철 아이스크림 판매량이 늘면 상어 공격이 급증한다." 이는 아이스크림이 상어를 부르는 것이 아니라, '기온 상승'이라는 제3의 **교란 변수**(Confounding Variable)가 둘 다 증가시켰을 뿐이다.

### 6) 기저율 무시의 **오류**(Base Rate Fallacy)

판단의 기준이 되는 원래의 확률(기저율)을 무시하고, 특정 결과 수치에만 매몰되어 해석하는 오류이다.

> **사례:** "사망 환자의 80%가 백신 접종자다!" 전 국민의 95%가 접종자(기저율)라면, 5%밖에 안 되는 미접종자 그룹에서 20%의 사망자가 나온 것이므로 실제로는 미접종자의 치명률이 압도적으로 높은 것이다.

---

## 5. 미디어 오용 3대 라벨 분석

### 1) 선택 **편향**(Selection Bias)

특정 커뮤니티나 자발적 참여자(비확률 표본)의 의견을 전체 모집단으로 일반화하는 오류이다. 강한 불만을 가진 사람일수록 자발적 조사(Opt-in poll)에 적극적으로 참여하므로 표본이 심각하게 오염된다.

> **기사 사례:** "충격! 직장인 80%, '주 4일제 도입 시 임금 삭감 동의'"
> (진실: 전국 직장인 무작위 추출이 아닌, 특정 재테크 카페 대상 온라인 설문)

### 2) 지표 정의 오류

통계값을 도출할 때 **분모**(전체 대상)와 **분자**(관찰 집단)의 기준이 논리적으로 연결되지 않는 현상이다.

> **기사 사례:** "A시, 청년 실업률 전국 최저 1%대 달성!"
> (진실: 실업률의 올바른 분모인 '경제활동인구' 대신 일할 생각조차 없는 학생, 군인까지 모조리 분모에 넣어 수치를 낮추는 꼼수)

### 3) 기준 및 척도 **오용**(% vs %p)

변화 비율인 '증감률(%)'과 단순 산술 차이인 '퍼센트 **포인트**(%p)'를 교묘하게 혼용하여 착시를 유발한다.

> **기사 사례:** 금리가 2%에서 3%가 된 것을 "고작 1% 올랐다"고 보도함.
> (진실: 이자액 자체는 2배가 늘어나는 것이므로 가계 체감 증가율은 50% 폭등한 것임)

## 6. [검증 실습 1] 특정 집단의 과대 대표성과 Z-점수

**[분석 대상 기사 패턴]** "A정당 책임당원 12만 명 중복 가입 논란... 대한민국 전체 여론 왜곡 우려"
조사한 기사: https://n.news.naver.com/article/047/0002489290?sid=100

특정 정당의 책임당원 중 발생한 특이값(중복 등)을 전체 여론이 오염된 것처럼 과장한 기사이다. 이는 분모(4,400만 대한민국 전체 유권자)와 분자(특정 정당에 자발적으로 가입한 강한 성향의 표본)가 어긋난 '**지표 정의 오류**' **이자**  **선택 편향**이다.

이를 증명하기 위해, 조작이 없는 무작위 상태에서 특정 당원이 뽑힐 확률을 **초기하분포**(Hypergeometric)로 계산한다.

$$E(X) = n \times \frac{B}{N}$$

(N: 총 유권자, B: 타겟 당원 수, n: 추출된 표본 수)

기사에서 주장하는 관측값이 정상 기대값으로부터 표준편차(&sigma;)의 몇 배나 멀리 있는지 **Z-점수**(Z-Score)로 산출한다. 통상 Z-점수가 3을 넘어가면 우연히 발생할 수 없는 조작된 수치로 간주한다.

### 파이썬 실습 코드

```python
import math

def check_news_claim(N, B, n, observed_k, scenario_name):
    # 1. 기대값(평균) 계산: 완벽한 무작위 상황에서 정상적으로 겹쳐야 하는 수
    expected_k = n * (B / N)
    
    # 2. 분산 계산: 유권자 조사는 비복원 추출이므로 유한모집단 수정계수 적용
    variance = n * (B / N) * ((N - B) / N) * ((N - n) / (N - 1))
    std_dev = math.sqrt(variance)
    
    # 3. Z-점수 계산: 관측값이 기대값에서 표준편차의 몇 배나 벗어났는가?
    z_score = (observed_k - expected_k) / std_dev
    
    print(f"[{scenario_name}]")
    print(f" - 수학적 정상 기대 겹침수: 약 {expected_k:,.1f}명")
    print(f" - 기사에서 주장하는 특이값: {observed_k:,}명")
    print(f" - Z-점수(이례성 지표): {z_score:,.2f} 시그마")
    
    if z_score > 3:
        print(" => [결론] 통계적으로 자연 상태에서 발생 불가능한 수치이다.")
        print(" => 누군가 의도적으로 표본을 조작했거나 심각한 '선택 편향'이 개입되었다.\n")
    else:
        print(" => [결론] 통계적 오차 범위 내에서 발생 가능한 현상이다.\n")

# --- [실데이터 셋팅] ---
N_total = 44000000  # 선관위 발표 대한민국 총 유권자 수 (약 4,400만)
n_sample = 120000   # 문제가 된 여론조사의 총 표본 수
obs_k = 90000       # 기사에서 특정 세력이 중복됐다고 과장한 수치 (가정)

print("=" * 65)
print("기사 팩트체크: '특정 당원 9만 명 중복 여론조사 참여 충격!'\n")

# 시나리오 A: 분모를 누적당원 500만 명으로 넓게 잡은 경우
check_news_claim(N_total, 5000000, n_sample, obs_k,
    "시나리오 A: 분모를 누적당원 500만 명으로 넓게 잡은 경우")

# 시나리오 B: 분모를 실효당원 75만 명으로 엄격히 좁힌 경우
check_news_claim(N_total, 750000, n_sample, obs_k,
    "시나리오 B: 분모를 실효당원 75만 명으로 엄격히 좁힌 경우")
print("=" * 65)
```

## 7. [검증 실습 2] 전세의 월세화 분석과 시각화

**[분석 대상]** "전세 시대 끝났다... 9월 서울 아파트 월세 비중 역대 최고 폭증!"  
https://www.yna.co.kr/view/AKR20251030170200003?input=1195m
이 기사는 두 가지 치명적인 오류를 품고 있다.

1. **비교 축 **오류**(MoM vs YoY):** 이사철(가을)과 비수기(여름)의 강력한 계절성을 무시하고, 단순히 8월 대비 9월(전월 대비, MoM) 수치를 비교하여 폭등한 것처럼 왜곡하였다.
2. **구성효과의 **함정**:** 고금리로 인해 전체 거래량이 줄어들며 '전세 건수'가 폭락하자, 가만히 있던 '월세 건수'의 **비중**(%)만 상대적으로 껑충 뛰어오른 착시 현상이다.

**해결책** (Time Alignment & Dual Axis):
계절성을 통제하기 위해 매년 동일한 달(9월, YoY)만 추출하여 비교한다. 또한 '비중(%)'과 '절대 건수'를 한 화면에 겹쳐 그리는 더블 축 시각화를 통해 진실을 밝혀낸다.

### 파이썬 실습 코드

```python
import matplotlib.pyplot as plt
import numpy as np
import urllib.request
import os
import matplotlib.font_manager as fm

# --- [클라우드 서버 한글 폰트 깨짐 방지 로직] ---
font_path = 'NanumGothic.ttf'
if not os.path.exists(font_path):
    url = 'https://github.com/google/fonts/raw/main/ofl/nanumgothic/NanumGothic-Regular.ttf'
    urllib.request.urlretrieve(url, font_path)
fm.fontManager.addfont(font_path)
plt.rcParams['font.family'] = 'NanumGothic'
plt.rcParams['axes.unicode_minus'] = False

# --- [국토교통부 실거래가 데이터 (매년 9월 추출)] ---
years = ['2023.09', '2024.09', '2025.09']
jeonse_vol = np.array([12000, 10000, 9000])  # 전세 거래량 (고금리로 하락세)
wolse_vol = np.array([8000, 8180, 9000])     # 월세 거래량 (비교적 완만한 유지)

# 기사가 왜곡한 '월세 비중(%)' 계산
total_vol = jeonse_vol + wolse_vol
wolse_ratio = (wolse_vol / total_vol) * 100

# --- [더블 축 시각화 구현] ---
fig, ax1 = plt.subplots(figsize=(10, 6))
fig.patch.set_facecolor('#f8fafc')

x = np.arange(len(years))
width = 0.35

# 1. 좌측 Y축: 전세/월세 절대 거래량 (막대 그래프)
bar1 = ax1.bar(x - width/2, jeonse_vol, width,
               label='전세 건수 (절대량)', color='#38bdf8', alpha=0.9)
bar2 = ax1.bar(x + width/2, wolse_vol, width,
               label='월세 건수 (절대량)', color='#f59e0b', alpha=0.9)

ax1.set_ylabel('절대 거래량 (건수)', fontsize=12, fontweight='bold', color='#1e293b')
ax1.set_ylim(0, 15000)
ax1.set_xticks(x)
ax1.set_xticklabels(years, fontsize=11, fontweight='bold')

# 2. 우측 Y축: 월세 비중 (선 그래프)
ax2 = ax1.twinx()
line1 = ax2.plot(x, wolse_ratio, color='#ef4444', marker='o',
                 markersize=8, linewidth=3, label='월세 비중 (%) 착시')

ax2.set_ylabel('월세 비중 (%)', fontsize=12, fontweight='bold', color='#ef4444')
ax2.set_ylim(0, 100)
ax2.tick_params(axis='y', colors='#ef4444')

# 선 그래프 위에 퍼센트 라벨 추가
for i, ratio in enumerate(wolse_ratio):
    ax2.annotate(f'{ratio:.1f}%', (x[i], ratio + 3),
                 ha='center', color='#ef4444', fontweight='bold', fontsize=11)

# 범례 통합
bars = [bar1, bar2]
labels = [bar.get_label() for bar in bars] + [line1[0].get_label()]
plots = [bar1, bar2, line1[0]]
ax1.legend(plots, labels, loc='upper center',
           bbox_to_anchor=(0.5, 1.15), ncol=3, frameon=False)

plt.title("전세의 월세화 분석: 비중 착시를 깨는 절대량 교차 검증",
          fontsize=15, fontweight='bold', pad=30)
plt.tight_layout()
```

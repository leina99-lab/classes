### 🎯 미션

뉴욕 에어비앤비 데이터에서 **"어디가 비싸고, 왜 비싼가?"**를 6단계 시각적 탐색으로 증명하라.

**규칙:**
- 각 단계마다 그래프를 그리고, **📝 관찰 노트**에 발견한 내용을 기록한다.
- 관찰에는 **반드시 구체적인 수치**($, %, 배수)를 포함한다.
- 모든 그래프에 **색맹 안전 팔레트**(`colorblind`, `viridis`, `cividis`)를 사용한다.
- "~때문이다"(인과)가 아닌 "~와 관련이 있다"(상관)로 표현한다.

```
❌ "Manhattan이 비싸다."  (일반론 — 0점)
✅ "Manhattan의 중앙값($150)은 Bronx($65)의 약 2.3배이며,
   IQR도 $95로 가장 넓어 가격 편차가 크다."  (수치 포함 — 합격)
```

### 1단계: 전체 가격 분포 — 히스토그램 + KDE

```python
# (A) 전체 분포와 (B) $300 이하 확대를 나란히 비교
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

sns.histplot(data=df_clean, x='price', bins=50, kde=True,
             ax=axes[0])
axes[0].set_title('전체 가격 분포')
axes[0].set_xlabel('가격 ($)')

df_under300 = df_clean[df_clean['price'] <= 300]
sns.histplot(data=df_under300, x='price', bins=30, kde=True,
             ax=axes[1])
axes[1].set_title('$300 이하 가격 분포 (확대)')
axes[1].set_xlabel('가격 ($)')

plt.tight_layout()
plt.show()
```

```python
# 기술통계 수치 확인
print(f"평균: ${df_clean['price'].mean():.0f}")
print(f"중앙값: ${df_clean['price'].median():.0f}")
print(f"최빈값: ${df_clean['price'].mode()[0]}")
print(f"왜도(skew): {df_clean['price'].skew():.2f}")
```

> 📝 **관찰 노트:**
> "전체 가격 분포는 오른쪽으로 ______(왜도=______)이며, 봉우리(최빈값)는 $______에 위치한다. 평균($______)이 중앙값($______)보다 높아, 고가 숙소가 평균을 끌어올리고 있다."

### 2단계: 지역별 분포 비교 — 박스플롯 + 바이올린

#### 2-A: 박스플롯

```python
order = (df_clean.groupby('neighbourhood_group')['price']
         .median().sort_values(ascending=False).index)

fig, axes = plt.subplots(1, 2, figsize=(14, 6))

# (A) 전체 박스플롯
sns.boxplot(data=df_clean, x='neighbourhood_group', y='price',
            order=order, palette="colorblind", ax=axes[0])
axes[0].set_title('지역별 가격 박스플롯')

# (B) $300 이하 확대
sns.boxplot(data=df_under300, x='neighbourhood_group', y='price',
            order=order, palette="colorblind", ax=axes[1])
axes[1].set_title('지역별 가격 ($300 이하 확대)')

plt.tight_layout()
plt.show()

# 지역별 기술통계
print(df_clean.groupby('neighbourhood_group')['price']
      .agg(['median', 'mean', 'std', 'count'])
      .sort_values('median', ascending=False)
      .round(0))
```

> 📝 **관찰 노트:**
> "중앙값이 가장 높은 지역은 ______(중앙값 $______), 가장 낮은 지역은 ______(중앙값 $______)이다. 두 지역의 중앙값 차이는 약 ______배이다. IQR이 가장 넓은 지역은 ______로, 가격 다양성이 크다."

#### 2-B: 바이올린 플롯 (심화)

```python
plt.figure(figsize=(12, 6))
sns.violinplot(data=df_under300,
               x='neighbourhood_group', y='price',
               order=order,
               palette="colorblind",
               inner='quartile',  # 사분위선 표시
               cut=0)             # 데이터 범위 밖 곡선 제거
plt.title('지역별 가격 바이올린 플롯 ($300 이하)')
plt.xlabel('지역')
plt.ylabel('가격 ($)')
plt.show()
```

> **바이올린 읽는 법:** 넓은 부분 = 그 가격대에 숙소가 많다. 봉우리가 2개 = **이봉분포**(bimodal), 가격대가 두 그룹으로 나뉨을 의미한다.
>
> 📝 **추가 관찰:** "______지역의 바이올린은 봉우리가 ______개이며, 이는 가격대가 ______그룹으로 나뉘어 있음을 시사한다."

### 4단계: room_type × 지역 교차 분석 — 히트맵

#### 4-A: 중앙 가격 히트맵

```python
# 교차표: 지역 × room_type의 중앙 가격
price_cross = pd.crosstab(
    index=df_clean['neighbourhood_group'],
    columns=df_clean['room_type'],
    values=df_clean['price'],
    aggfunc='median'
).round(0)

plt.figure(figsize=(8, 5))
sns.heatmap(price_cross,
            annot=True, fmt='.0f',
            cmap='viridis',            # ⭐ 색맹 안전!
            linewidths=0.5,
            cbar_kws={'label': '중앙 가격 ($)'})
plt.title('지역 × 숙소 유형별 중앙 가격 ($)')
plt.ylabel('지역')
plt.xlabel('숙소 유형')
plt.show()
```

> 📝 **관찰 노트:** "가장 비싼 조합은 ______ × ______으로 중앙 가격 $______이다. 같은 room_type이라도 지역에 따라 최대 ______배 차이가 난다."

#### 4-B: 매물 수 히트맵 (비교용)

```python
count_cross = pd.crosstab(
    index=df_clean['neighbourhood_group'],
    columns=df_clean['room_type']
)

fig, axes = plt.subplots(1, 2, figsize=(16, 5))

sns.heatmap(count_cross, annot=True, fmt='d',
            cmap='cividis', ax=axes[0])    # ⭐ cividis: 색맹 전용 설계!
axes[0].set_title('지역 × 유형별 매물 수')

sns.heatmap(price_cross, annot=True, fmt='.0f',
            cmap='viridis', ax=axes[1])
axes[1].set_title('지역 × 유형별 중앙 가격 ($)')

plt.tight_layout()
plt.show()
```

> 💡 **두 히트맵을 나란히 보는 이유:** 가격이 높다고 매물이 많은 것은 아니다. "비싸면서 매물도 많은 조합"과 "비싸지만 매물이 적은 조합"은 의미가 다르다.
>

### 5단계: 숫자 변수 상관 분석 — 히트맵 + 산점도

#### 5-A: 상관 히트맵

```python
num_cols = ['price', 'minimum_nights', 'number_of_reviews',
            'reviews_per_month', 'calculated_host_listings_count',
            'availability_365']

corr = df_clean[num_cols].corr()

plt.figure(figsize=(9, 7))
sns.heatmap(corr, annot=True, fmt='.2f',
            cmap='coolwarm', center=0,
            vmin=-1, vmax=1,
            linewidths=0.5, square=True)
plt.title('숫자 변수 간 상관계수 히트맵')
plt.tight_layout()
plt.show()
```

#### 5-B: 상관이 가장 높은 변수 쌍의 산점도

```python
# 가격과의 상관계수 (절대값 내림차순)
price_corr = corr['price'].drop('price').abs().sort_values(ascending=False)
print("가격과의 상관계수:")
print(price_corr.round(3))

# 상위 2개 변수와의 산점도
top2 = price_corr.index[:2]
fig, axes = plt.subplots(1, 2, figsize=(14, 5))
for i, col in enumerate(top2):
    sns.regplot(data=df_clean, x=col, y='price',
                scatter_kws={'alpha': 0.1, 's': 5},
                line_kws={'color': 'red'},
                ax=axes[i])
    r = corr.loc['price', col]
    axes[i].set_title(f'가격 vs {col} (r={r:.3f})')
plt.tight_layout()
plt.show()
```

> 📝 **관찰 노트:** "가격(price)과 가장 높은 상관을 보이는 변수는 ______(r=______)이다. 다만 │r│< 0.3이므로 에어비앤비 가격은 단일 숫자 변수로는 설명하기 어렵다. 위치(지역)와 숙소 유형 같은 **범주형 변수**가 더 중요한 것으로 보인다."

### 1.1 왜 시각화인가? — **앤스콤의 사중주**(Anscombe's Quartet)

1973년, 통계학자 **프랭크 앤스콤**(Frank Anscombe)은 충격적인 데이터를 발표하였다. 네 개의 데이터셋이 있는데, 통계량이 **완전히 동일**하다.

| 통계량 | 데이터셋 I | 데이터셋 II | 데이터셋 III | 데이터셋 IV |
|---|---|---|---|---|
| x **평균**(mean) | 9.0 | 9.0 | 9.0 | 9.0 |
| y **평균**(mean) | 7.50 | 7.50 | 7.50 | 7.50 |
| x **표준편차**(std) | 3.32 | 3.32 | 3.32 | 3.32 |
| y **표준편차**(std) | 2.03 | 2.03 | 2.03 | 2.03 |
| **상관계수**(r) | 0.816 | 0.816 | 0.816 | 0.816 |
| **회귀식** | y=3+0.5x | y=3+0.5x | y=3+0.5x | y=3+0.5x |

평균, 표준편차, 상관계수, 회귀식까지 전부 같다. 그렇다면 이 네 데이터는 "같다"고 할 수 있는가?

**정답: 완전히 다르다.**

```
  데이터셋 I         데이터셋 II        데이터셋 III       데이터셋 IV
   (선형)            (곡선)            (이상치 1개)       (이상치 극단)

    ·  ·              · · ·                 ·  ·              ·
   · ·  ·           ·       ·             · · ·              ·
  ·  · ·           ·         ·           · · ·              ·
 ·  ·             ·           ·         ·  ·               ·
  ·              ·             ·       ·                   ·
                                    ★(이상치)         ★───────·
```

- **데이터셋 I**: 교과서적인 선형 관계이다.
- **데이터셋 II**: 실제로는 곡선(포물선) 관계이다. 직선 모델은 완전히 잘못된 것이다.
- **데이터셋 III**: 거의 완벽한 직선이지만, **이상치 1개**가 회귀선을 끌어당기고 있다.
- **데이터셋 IV**: x값이 전부 8인데 딱 하나만 19이다. 그 한 점 때문에 상관계수 0.816이 산출된 것이며, 나머지 10개 점은 수직선 위에 있으므로 **상관관계 자체가 무의미**하다.

> 💡 **앤스콤의 교훈**: "항상 그래프를 먼저 그려라."
> 통계량만으로는 데이터의 진짜 모습을 알 수 없다.

### 1.2 **탐색적 데이터 분석**(EDA)이란?

**EDA**(Exploratory Data Analysis)란, 데이터를 본격적으로 분석하기 전에 그래프와 통계량을 사용하여 데이터의 구조, 분포, 관계를 탐색하는 과정이다. 1977년 통계학자 **존 튜키**(John Tukey)가 체계화하였다.

```
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ 1단계     │    │ 2단계     │    │ 3단계     │
  │ 구조 파악  │ →  │ 분포 확인  │ →  │ 관계 탐색  │
  │          │    │          │    │          │
  │ .shape   │    │ 히스토그램 │    │ 산점도    │
  │ .info()  │    │ 박스플롯   │    │ 상관 히트맵 │
  │ .dtypes  │    │ describe()│    │ 그룹 비교  │
  └──────────┘    └──────────┘    └──────────┘
     4주차           ←── 5주차 ──→
```

1단계는 4주차에서 이미 완료하였다. 5주차에서는 2단계(분포 확인)와 3단계(관계 탐색)에 집중한다.

> 💡 EDA의 목적은 "답을 찾는 것"이 아니라 **"좋은 질문을 찾는 것"**이다.

### 1.3 기술통계 복습과 심화

#### 1.3.1 **중심 경향**(Central Tendency) — 평균 vs 중앙값

**평균**(Mean)은 모든 값을 더해 개수로 나눈 것이다. **중앙값**(Median)은 값을 정렬했을 때 한가운데 위치한 값이다.

```
예시: 5명의 월급 (단위: 만원)

  200, 250, 300, 280, 270
  → 평균 = 260, 중앙값 = 270  ← 비슷하다

  200, 250, 300, 280, 5000  ← 사장님 등장!
  → 평균 = 1,206, 중앙값 = 280  ← 완전히 다르다!
```

이상치가 있으면 평균이 크게 왜곡된다. 에어비앤비 가격처럼 **오른쪽 꼬리가 긴 데이터**에서는 중앙값이 더 신뢰할 수 있는 대표값이다.

#### 1.3.2 **산포도**(Spread) — 표준편차

**표준편차**(Standard Deviation, std)는 각 값이 평균에서 평균적으로 얼마나 떨어져 있는지를 측정한다.

```
  반 A: 70, 80, 75, 85, 90  → 평균 80, std ≈ 7.1   (좁게 모임)
  반 B: 50, 60, 80, 100, 110 → 평균 80, std ≈ 23.5  (넓게 흩어짐)

  반 A:           ▓▓▓▓▓
              ──┼─────────── (평균 80)

  반 B:   ▓▓▓          ▓▓▓▓▓▓
       ──────┼───────────────── (평균 80)
```

평균이 같아도 표준편차가 다르면 데이터의 "모양"이 완전히 다르다.

#### 1.3.3 **사분위수**(Quartile)와 **박스플롯**(Box Plot)

사분위수는 데이터를 크기순으로 정렬한 뒤 4등분하는 세 개의 지점이다.

- **Q1**(25번째 백분위): 하위 25% 지점
- **Q2**(50번째 백분위): 중앙값
- **Q3**(75번째 백분위): 상위 25% 지점
- **IQR**(사분위 범위) = Q3 − Q1: 데이터의 "중간 50%"가 차지하는 범위

```
박스플롯 구조도:

  이상치   수염         상자            수염   이상치
    ○ ────┤     ┌──────┬──────┐     ├──── ○
          │     │      │      │     │
       Q1−1.5×IQR  Q1   Q2    Q3  Q3+1.5×IQR

  ┌────────────────────────────────────────────┐
  │ 상자(Box) = Q1~Q3 = 데이터의 중간 50%        │
  │ 가운데 선 = Q2 = 중앙값                       │
  │ 수염(Whisker) = Q1−1.5×IQR ~ Q3+1.5×IQR     │
  │ 수염 바깥 = ○ = 이상치(Outlier)               │
  └────────────────────────────────────────────┘
```

박스플롯 하나로 중앙값, 퍼짐 정도, 이상치를 **동시에** 볼 수 있다.

#### 1.3.4 **상관계수**(Correlation Coefficient)

두 변수 사이의 **선형 관계 강도**를 −1에서 +1 사이의 숫자로 요약한 것이다.

```
  r ≈ +1.0     r ≈ 0       r ≈ −1.0
   · ·/         · · ·         \· ·
  · /·         ·  ·  ·        ·\ ·
 ·/ ·          · · · ·        · \·
  강한 양(+)    관계 없음      강한 음(−)
```

| 범위 (절대값) | 해석 |
|---|---|
| │r│ < 0.3 | 약한 상관 |
| 0.3 ≤ │r│ < 0.7 | 보통 상관 |
| │r│ ≥ 0.7 | 강한 상관 |

> ⚠️ **상관**(Correlation) **≠ 인과**(Causation)
>
> 아이스크림 판매량이 올라가면 익사 사고도 올라간다(r ≈ +0.8). 아이스크림이 익사를 일으킨 것인가? 아니다. **여름(기온)**이라는 숨은 변수가 둘 다에 영향을 준 것이다. 상관계수가 높아도 "A가 B를 일으킨다"라고 말할 수 없다. "A와 B는 관련이 있다"라고 표현해야 한다.

### 1.4 어떤 그래프를 골라야 하는가? — 유형별 선택 가이드

```
 ┌────────────────────────────────────────────┐
 │            내가 보고 싶은 것은?               │
 └────────────┬──────────────┬────────────────┘
              │              │
     ┌────────▼──────┐  ┌───▼──────────┐
     │ 하나의 변수     │  │ 두 변수의 관계 │
     │ (분포가 궁금)   │  │ (패턴이 궁금)  │
     └───┬──────┬────┘  └──┬──────┬────┘
         │      │          │      │
    ┌────▼──┐ ┌▼─────┐ ┌──▼───┐ ┌▼──────┐
    │히스토  │ │박스   │ │산점도 │ │히트맵  │
    │그램    │ │플롯   │ │      │ │       │
    └───────┘ └──────┘ └──────┘ └───────┘
```

| 질문 | 그래프 | 설명 |
|---|---|---|
| 값의 분포를 자세히 보고 싶다 | **히스토그램** + **KDE** | 구간별 빈도 + 부드러운 밀도 곡선 |
| 여러 그룹의 분포를 비교하고 싶다 | **박스플롯** / **바이올린** | 중앙값, IQR, 이상치를 한눈에 |
| 두 숫자 변수의 관계가 궁금하다 | **산점도** | 점 하나 = 데이터 한 행 |
| 여러 변수의 상관을 한눈에 보고 싶다 | **히트맵** | 색상 강도 = 상관 강도 |
| 범주별 개수를 비교하고 싶다 | **막대그래프**(Count Plot) | 범주형 변수의 빈도 |

### 1.5 **색상의 과학** — 색맹 배려와 전문가 팔레트 ⭐

#### 1.5.1 왜 색맹을 고려해야 하는가?

**색각 이상**(Color Vision Deficiency, CVD)은 특정 색상을 구분하기 어려운 조건이다.

- 전 세계 **남성의 약 8%,** **여성의 약 0.5%**가 해당한다.
- 30명 반에 남학생이 15명이면, 통계적으로 **약 1.2명**이 색각 이상이다.
- 가장 흔한 유형: **적록 색맹**(Red-Green CVD) — 빨강과 초록을 구분하기 어렵다.

```
일반 시각        적록 색맹이 보는 세계
┌──────────┐    ┌──────────┐
│ 🔴 빨강    │    │ 🟤 갈색    │
│ 🟢 초록    │    │ 🟤 갈색    │  ← 구분 불가!
│ 🔵 파랑    │    │ 🔵 파랑    │
│ 🟡 노랑    │    │ 🟡 노랑    │
└──────────┘    └──────────┘
```

빨강과 초록을 함께 사용하면, 적록 색맹인 사람은 두 그룹을 **구분할 수 없다.**

> 💡 데이터 시각화는 "예쁜 그래프"가 아니라 **"모든 사람이 읽을 수 있는 그래프"**를 만드는 것이다.

#### 1.5.2 색맹 안전 팔레트(CVD-Safe Palettes)

**✅ 추천:**

| 팔레트 | 용도 | 코드 |
|---|---|---|
| **colorblind** | 범주형 색상 (8색) | `sns.set_palette("colorblind")` |
| **viridis** | 연속값 (보라→초록→노랑) | `cmap='viridis'` |
| **cividis** | 연속값 (파랑→노랑, 색맹 전용 설계) | `cmap='cividis'` |
| **파랑+주황** | 2그룹 비교 시 가장 안전 | 기본값에 포함 |

**❌ 피해야 하는 것:**

| 팔레트 | 이유 |
|---|---|
| **jet / rainbow** | 적록 색맹에게 중간 구간 구분 불가 |
| **빨강 + 초록** 조합 | 적록 색맹에게 동일하게 보임 |

#### 1.5.3 색상 외 추가 구분 수단

색상에만 의존하면 흑백 인쇄 시에도 구분이 불가능하다. **전문가의 전략**은 다음을 병행하는 것이다.

- **마커 모양** 변경: ○ □ △ ◇ → `style=` 파라미터
- **선 스타일** 변경: 실선, 점선, 파선
- **직접 라벨** 표시: 그래프 위에 텍스트 배치

```python
# 색상 + 모양을 함께 사용하는 예시
sns.scatterplot(x='reviews', y='price',
                hue='borough',    # → 색상으로 구분
                style='borough',  # → 마커 모양으로도 구분
                data=df)
```

> ⭐ **오늘부터의 습관**: 모든 노트북의 첫 셀에 `sns.set_palette("colorblind")`를 넣는다.

### 1.6 도구 비교: Matplotlib vs Seaborn vs Plotly

```
                  Plotly (대화형)
                     │
  ┌─────────────────────────────────────┐
  │          Seaborn (통계 시각화)        │

  │  ┌──────────────────────────────┐   │

  │  └──────────────────────────────┘   │
  └─────────────────────────────────────┘
```

|---|---|---|---|
| 코드 길이 | 길다 | 짧다 | 중간 |

| **대화형** | ❌ | ❌ | ✅ |
| 주 용도 | 세부 커스터마이징 | EDA · 논문 | 발표 · 웹 |

### 1.7 Seaborn 핵심 문법

```python
import seaborn as sns
import matplotlib.pyplot as plt

# 1단계: 팔레트 & 스타일
sns.set_palette("colorblind")
sns.set_style("whitegrid")

# 2단계: 그래프 그리기
sns.histplot(data=df, x='price', kde=True)

# 3단계: 제목 & 라벨
plt.title('에어비앤비 가격 분포')
plt.xlabel('가격 ($)')
plt.ylabel('빈도')
plt.show()
```

**자주 쓰는 함수:**

| 함수 | 용도 |
|---|---|
| `sns.histplot()` | 히스토그램 + KDE |
| `sns.boxplot()` | 박스플롯 |
| `sns.violinplot()` | 바이올린플롯 |
| `sns.scatterplot()` | 산점도 |
| `sns.heatmap()` | 히트맵 |
| `sns.countplot()` | 카운트(범주별 빈도) |
| `sns.regplot()` | 산점도 + 회귀선 |
| `sns.pairplot()` | 모든 변수 쌍 산점도 |

**핵심 파라미터:** `data=`(DataFrame), `x=`/`y=`(변수), `hue=`(색상 그룹), `style=`(모양 그룹, 색맹 대비), `palette=`(팔레트)

### 실습 3: 지역별 가격 박스플롯 (색맹 안전)

```python
order = (df_clean.groupby('neighbourhood_group')['price']
         .median().sort_values(ascending=False).index)

plt.figure(figsize=(10, 6))
sns.boxplot(data=df_clean, x='neighbourhood_group', y='price',
            order=order,
            palette="colorblind")      # ⭐ 색맹 안전!
plt.title('지역별 숙박 가격 분포')
plt.xlabel('지역')
plt.ylabel('가격 ($)')
plt.show()
```

> 📝 **박스플롯 읽는 순서:**
> 1. 상자 위치 → 중앙값 비교
> 2. 상자 길이 → 퍼짐(IQR) 비교
> 3. 이상치(○) → 초고가 숙소 확인

### 실습 5: 상관 히트맵

```python
num_cols = ['price', 'minimum_nights', 'number_of_reviews',
            'reviews_per_month', 'calculated_host_listings_count',
            'availability_365']
corr = df_clean[num_cols].corr()

plt.show()
```

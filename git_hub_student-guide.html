Git 기초 명령어

과제 제출에 필요한 Git 명령어 4가지만 알면 됩니다.

clone → 코드 작성 → add → commit → push → 자동 채점 ✓
1 git clone — 과제 저장소를 내 컴퓨터로 복사

초대 링크를 통해 만들어진 내 저장소(repo)를 로컬로 다운로드합니다.

CMD / Terminal git clone https://github.com/ORG이름/과제이름-내아이디.git
CMD / Terminal cd 과제이름-내아이디
✅
저장소 URL은 GitHub에서 "Code" 버튼을 눌러 복사할 수 있습니다.
2 git add — 변경한 파일을 제출 목록에 추가

코드를 작성한 뒤, 변경된 파일을 Git에 알려줍니다.

CMD / Terminal # 모든 변경 파일 추가 git add . # 또는 특정 파일만 추가 git add assignment.ipynb
3 git commit — 변경 내용을 저장 (메시지와 함께)

무엇을 변경했는지 메시지를 남기며 저장합니다.

CMD / Terminal git commit -m "문제 1, 2 풀이 완료"
⚠️
commit 메시지는 따옴표 안에 작성해야 합니다. 한글도 괜찮습니다.
4 git push — GitHub에 업로드 (제출!)

로컬의 변경 사항을 GitHub 서버로 보냅니다. push를 해야 자동 채점이 시작됩니다.

CMD / Terminal git push origin main
✅
마감 전까지 여러 번 push할 수 있습니다. 마지막 push가 최종 제출이 됩니다.
명령어 뜻 비유
git clone 저장소 복사 📋 시험지 받기
git add . 변경 파일 선택 ✏️ 답안지에 이름 쓰기
git commit -m "메시지" 변경 내용 저장 📦 답안지 봉투에 넣기
git push GitHub에 업로드 📮 답안지 제출하기

Python Notebook 과제 제출

초대 링크 받기부터 자동 채점 결과 확인까지

1 초대 링크 클릭 → 과제 수락

교수님이 공유한 초대 링크를 클릭합니다.

처음이라면 본인 이름을 선택하고 → "Accept this assignment" 버튼을 클릭합니다.

⚠️
이름을 잘못 선택하면 나중에 성적이 꼬일 수 있으니 반드시 본인 이름을 선택하세요!
2 내 저장소가 자동 생성됨

수락하면 나만의 저장소가 만들어지고, 링크가 표시됩니다.

예: https://github.com/수업-org/과제1-홍길동

3 저장소를 내 컴퓨터로 clone
CMD / Terminal git clone https://github.com/수업-org/과제1-홍길동.git cd 과제1-홍길동
4 assignment.ipynb 열어서 문제 풀기

Jupyter Notebook 또는 VS Code에서 assignment.ipynb 파일을 엽니다.

CMD / Terminal # Jupyter로 열기 jupyter notebook assignment.ipynb # 또는 VS Code로 열기 code assignment.ipynb

각 문제의 pass를 지우고 코드를 작성하세요.

Python # ❌ 변경 전 def calculate_stats(data): pass # ✅ 변경 후 def calculate_stats(data): mean = np.mean(data) std = np.std(data) return (mean, std)
5 제출하기 (add → commit → push)
CMD / Terminal git add . git commit -m "과제 풀이 완료" git push origin main
✅
마감 전까지 몇 번이든 다시 push할 수 있습니다. 부분적으로 풀어도 먼저 push해 두는 것을 추천합니다.
6 자동 채점 결과 확인

push 후 GitHub에서 내 저장소 페이지에 들어갑니다.

방법 1: 저장소 메인 페이지 상단에 ✅ (초록) 또는 ❌ (빨강) 아이콘이 표시됩니다.

방법 2: "Actions" 탭을 클릭하면 자세한 채점 결과를 볼 수 있습니다.

아이콘 의미
✅ 모든 테스트 통과 — 만점!
❌ 일부 또는 전체 테스트 실패 — Actions 탭에서 어떤 문제가 틀렸는지 확인
🟡 채점 진행 중 — 1~2분 기다리세요

GitHub Codespaces로 과제하기 선택사항

Git, Python, Jupyter를 설치하지 않고 브라우저에서 바로 과제를 할 수 있습니다.

1 Codespace 만들기

내 저장소 페이지에서:

Code 버튼 클릭 → Codespaces 탭 → Create codespace on main

잠시 기다리면 브라우저에서 VS Code가 열립니다.

2 노트북 열어서 문제 풀기

왼쪽 파일 탐색기에서 assignment.ipynb를 클릭하면 노트북이 열립니다.

셀을 클릭하고 코드를 작성한 뒤, Shift + Enter로 실행할 수 있습니다.

3 Codespace에서 제출하기

Codespace 안에 터미널이 내장되어 있습니다. 상단 메뉴 Terminal → New Terminal 클릭 후:

Terminal git add . git commit -m "과제 풀이 완료" git push origin main

또는 왼쪽 사이드바의 소스 제어 (Source Control) 아이콘을 이용해서 GUI로도 제출할 수 있습니다:

소스 제어 아이콘 클릭 → + 버튼 (Stage) → 메시지 입력 → ✓ Commit → Sync Changes
⚠️
Codespaces 무료 사용량 주의
GitHub 무료 계정 기준 월 120시간 / 15GB 저장공간입니다. 사용하지 않을 때는 Codespace를 Stop하세요. (Codespace 목록에서 ··· → Stop 클릭)

자주 묻는 질문

문제가 생겼을 때 여기를 먼저 확인하세요.

push했는데 자동 채점이 안 돼요

Actions 탭에 들어가서 워크플로우 실행 상태를 확인하세요. "Autograding"이 보이지 않으면 교수님께 문의하세요. 간혹 GitHub 서버 지연으로 1~2분 걸릴 수 있습니다.

git push가 안 돼요 (rejected 에러)

먼저 git pull origin main을 실행한 뒤 다시 push해 보세요. 다른 곳에서 변경한 내용이 있으면 이 에러가 발생합니다.

GitHub 로그인 창이 계속 뜨는데요?

Git Credential Manager를 설정하면 매번 로그인하지 않아도 됩니다. Windows에서는 Git 설치 시 함께 설치됩니다. 또는 GitHub CLI (gh auth login)를 사용하세요.

테스트는 통과했는데 점수가 안 나와요

GitHub Classroom 대시보드에서 점수가 반영되기까지 시간이 걸릴 수 있습니다. Actions 탭에서 ✅가 보이면 정상적으로 통과한 것입니다.

마감 후에도 push할 수 있나요?

기술적으로 push는 가능하지만, 교수님 설정에 따라 마감 이후 제출은 인정되지 않을 수 있습니다. 반드시 마감 전에 제출하세요.

Codespace가 안 만들어져요

무료 사용량을 초과했을 수 있습니다. github.com/settings/billing에서 사용량을 확인하세요. 초과 시 로컬에서 작업해야 합니다.

문제가 해결되지 않으면 교수님께 문의하세요.
